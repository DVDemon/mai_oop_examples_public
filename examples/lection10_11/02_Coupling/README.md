# Пример: Снижение связанности (Reduced Coupling)

## Описание

Данный пример демонстрирует **снижение связанности** по сравнению с примером `01_Coupling` через инкапсуляцию логики вычислений.

## Суть примера

В отличие от `01_Coupling`, где класс `Order` напрямую обращался к полям структур, здесь:

- **CartEntry** инкапсулирует вычисление стоимости позиции в методе `GetLineItemTotal()`
- **CartContents** инкапсулирует вычисление суммы корзины в методе `GetCartItemsTotal()`
- **Order** использует публичные методы вместо прямого доступа к полям

## Улучшение

Снижение связанности достигается за счет:
- **Инкапсуляции** - логика вычислений скрыта внутри структур
- **Абстракции** - класс `Order` работает с интерфейсом, а не с реализацией
- **Разделения ответственности** - каждая структура отвечает за свои вычисления

## Преимущества

- **Меньшая зависимость** - изменения в структуре данных не требуют изменений в `Order`
- **Лучшая поддерживаемость** - логика вычислений централизована
- **Гибкость** - можно изменить реализацию методов без влияния на `Order`

## Сравнение

Сравните с примером `01_Coupling`, чтобы увидеть разницу в уровне связанности.
