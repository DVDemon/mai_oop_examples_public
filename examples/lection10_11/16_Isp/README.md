# Пример: Нарушение принципа разделения интерфейсов (ISP)

## Описание

Данный пример демонстрирует **нарушение принципа разделения интерфейсов (Interface Segregation Principle, ISP)** - класс `CoffeMenu` вынужден реализовывать метод `GetCompot()`, который ему не нужен.

## Суть примера

Интерфейс `IBusinessMenu` содержит три метода:
- `GetFirstItem()` - первое блюдо
- `GetSecondItem()` - второе блюдо
- `GetCompot()` - компот

### Проблема

Класс `CoffeMenu` наследуется от `IBusinessMenu`, но:
- Ему не нужен метод `GetCompot()` (кофейное меню не включает компот)
- Вынужден реализовывать его, возвращая `nullptr`
- Нарушает принцип ISP

```cpp
class CoffeMenu : public IBusinessMenu {
    const char* GetCompot() override { 
        return nullptr;  // Ненужный метод!
    }
};
```

## Принцип ISP

**Клиенты не должны зависеть от интерфейсов, которые они не используют.**

Интерфейсы должны быть:
- **Специфичными** - содержать только необходимые методы
- **Разделенными** - лучше несколько маленьких интерфейсов, чем один большой
- **Без принуждения** - классы не должны реализовывать ненужные методы

## Проблемы текущей реализации

- **Принуждение к реализации** - `CoffeMenu` должен реализовывать ненужный метод
- **Нарушение семантики** - возврат `nullptr` указывает на проблему дизайна
- **Слабая переиспользуемость** - нельзя использовать только нужные части интерфейса
- **Сложность поддержки** - изменения в интерфейсе затрагивают все классы
