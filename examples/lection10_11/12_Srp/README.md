# Пример: Расширяемость через разделение ответственностей (SRP)

## Описание

Данный пример демонстрирует **расширяемость решения** при правильном применении принципа единственной ответственности (SRP) - возможность добавления новых типов передачи данных без изменения существующих интерфейсов.

## Суть примера

В дополнение к примеру `11_Srp`, где был интерфейс `ICharModem` для передачи символов, здесь:

### Добавлен новый интерфейс
```cpp
class IIntModem {
    // Передача целых чисел: Send, Receive, NotEmpty
};
```

### Гибкая реализация
```cpp
class Modem : public IConnection, public IIntModem {
    // Может использовать любой интерфейс передачи данных
};
```

## Преимущества разделения

- **Расширяемость** - можно добавить новые типы передачи данных (`ICharModem`, `IIntModem`, и т.д.)
- **Открытость для расширения** - новые интерфейсы не требуют изменения существующих
- **Гибкость использования** - класс `Modem` может реализовать любой набор интерфейсов
- **Независимость тестирования** - функции `Test()` перегружены для разных типов модемов

## Принцип открытости/закрытости (OCP)

Этот пример также демонстрирует принцип **Open/Closed Principle**:
- **Открыт для расширения** - можно добавлять новые интерфейсы
- **Закрыт для модификации** - существующие интерфейсы не изменяются
