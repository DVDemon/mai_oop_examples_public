# Пример: Разделение ответственностей через интерфейсы (SRP)

## Описание

Данный пример демонстрирует **правильное применение принципа единственной ответственности (SRP)** через разделение ответственностей на отдельные интерфейсы.

## Суть примера

В отличие от примеров `09_Srp` и `10_Srp`, где класс `Modem` нарушал SRP, здесь ответственности разделены:

### Интерфейс управления соединением
```cpp
class IConnection {
    // Управление соединением: Deal, Connected, Hangup
};
```

### Интерфейс передачи данных
```cpp
class ICharModem {
    // Передача символов: Send, Receive, NotEmpty
};
```

### Реализация
```cpp
class Modem : public IConnection, public ICharModem {
    // Просто наследует оба интерфейса
};
```

## Преимущества

- **Разделение ответственностей** - каждая ответственность в своем интерфейсе
- **Независимость изменений** - изменения в логике соединения не влияют на передачу данных
- **Гибкость** - можно использовать только нужный интерфейс
- **Тестируемость** - функции `Connect()`, `Test()`, `Disconnect()` работают с интерфейсами

## Принцип SRP

Каждый интерфейс имеет **только одну ответственность**:
- `IConnection` - только управление соединением
- `ICharModem` - только передача символов

Сравните с примерами `09_Srp` и `10_Srp`, чтобы увидеть разницу между нарушением и соблюдением SRP.
