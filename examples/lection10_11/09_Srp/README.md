# Пример: Нарушение принципа единственной ответственности (SRP)

## Описание

Данный пример демонстрирует проблему **нарушения принципа единственной ответственности (Single Responsibility Principle, SRP)** - класс `Modem` имеет несколько причин для изменения.

## Суть примера

Класс `Modem` объединяет две различные ответственности:

### 1. Управление соединением
- `Deal()` - установка соединения
- `Connected()` - проверка состояния соединения
- `Hangup()` - разрыв соединения

### 2. Передача данных
- `Send()` - отправка данных
- `Receive()` - получение данных

## Проблема

Нарушение SRP приводит к:
- **Множественные причины для изменения** - изменения в логике соединения или передачи данных требуют модификации одного класса
- **Сложность тестирования** - необходимо тестировать обе ответственности вместе
- **Слабая переиспользуемость** - нельзя использовать передачу данных без управления соединением
- **Высокая связанность** - логика соединения и передачи данных тесно переплетены

## Принцип SRP

Класс должен иметь **только одну причину для изменения** - только одну ответственность.
