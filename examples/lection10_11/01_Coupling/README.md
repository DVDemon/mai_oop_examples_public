# Пример: Связанность (Coupling)

## Описание

Данный пример демонстрирует проблему **тесной связанности (tight coupling)** в объектно-ориентированном программировании.

## Суть примера

Класс `Order` тесно связан со структурой `CartContents`:
- Прямой доступ к внутреннему полю `items` структуры `CartContents`
- Прямой доступ к полям структуры `CartEntry` (`Price`, `Quantity`)
- Любые изменения в структуре данных требуют изменений в классе `Order`

## Проблема

Тесная связанность снижает:
- **Гибкость** - сложно изменить структуру данных
- **Переиспользуемость** - класс нельзя использовать с другими типами корзин
- **Поддерживаемость** - изменения в одной части требуют изменений в других
