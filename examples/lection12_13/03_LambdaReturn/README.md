# Пример: Возвращаемый тип лямбда-выражений

## Описание

Данный пример демонстрирует **явное указание возвращаемого типа** в лямбда-выражениях с помощью синтаксиса `-> тип`.

## Суть примера

### Лямбда с явным возвращаемым типом

```cpp
auto transform = [](int i) -> double {
    if (i < 5) return i + 1.0;
    else if (i % 2 == 0) return i / 2.0;
    else return i * i;
};
```

### Синтаксис

```
[захват](параметры) -> возвращаемый_тип { тело }
```

В данном примере:
- `[]` - **пустой захват** - лямбда не захватывает переменные
- `(int i)` - параметр типа `int`
- `-> double` - **явное указание возвращаемого типа**
- `{ ... }` - тело с условной логикой

## Когда нужен явный тип возврата

### Автоматический вывод типа

В простых случаях компилятор может вывести тип автоматически:

```cpp
auto lambda = [](int x) { return x * 2; };  // Тип: int
```

### Явное указание типа

Явное указание необходимо когда:
- **Разные типы в return** - разные ветки возвращают разные типы
- **Нужна точность** - требуется конкретный тип (например, `double` вместо `int`)
- **Сложная логика** - компилятор не может вывести тип

В данном примере все ветки возвращают `double`, но явное указание делает код более понятным.

## Применение

Лямбда `transform` применяется к массиву для преобразования значений:

```cpp
for (int i = 0; i < 10; i++) 
    results[i] = transform(elements[i]);
```

Логика преобразования:
- Если `i < 5`: возвращает `i + 1.0`
- Если `i` четное: возвращает `i / 2.0`
- Иначе: возвращает `i * i`

## Преимущества явного типа

- **Ясность** - сразу видно, какой тип возвращается
- **Контроль** - гарантия правильного типа результата
- **Документация** - тип служит документацией для читателя кода
- **Избежание ошибок** - предотвращает неожиданные преобразования типов

## Сравнение

**Без явного типа:**
```cpp
auto lambda = [](int i) {
    return i * 2.0;  // Компилятор выведет double
};
```

**С явным типом:**
```cpp
auto lambda = [](int i) -> double {
    return i * 2.0;  // Явно указан double
};
```
